重要文件夹和文件

创建好hexo项目后，有几个重要的目录和文件

  1. scaffolds: 模板文件夹。 当您新建文章时，Hexo 会根据 scaffold 来创建文件。
  2. source: 资源文件夹。 是存放用户资源的地方。 除 _posts 文件夹之外，开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。 Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。
  3. themes: 主题文件夹。下载的主题存放在这里面
  4. _config.yml: 网站的配置文件。 您可以在此配置大部分的参数。
  5. package.json: 应用程序的信息。 EJS, Stylus 和 Markdown 渲染引擎 已默认安装，您可以自由移除。
  6. public：存储程序构建运行的文件。

Front-mater

Front-matter 是文件开头的 YAML 或 JSON 代码块，用于配置写作设置

布局和创建

hexo默认有三种布局模板，post、page、draft。

布局 路径
post source/_posts
page source
draft source/_drafts

文章（post）

默认的创建是post（可以在配置文件设置default_layout）。

创建的文件在source/_posts/xxx。

页面(page)

会在source文件夹里创建出一个xxx的文件夹，文件夹里有index.md文件。

可以通过访问地址 localhost:4000/xxx 访问的到页面。一般用来弄详情页。

草案(draft)

draft就是草案，草稿的意思。使用draft创建时，会创建在source/_draft文件夹里。上面说过了，source文件夹里，除了_post文件夹外，其他的开头带_的文件夹和文件会被忽略，不会编译到public文件夹里，所以也不会发布部署了。

虽然hexo不会编译draft的文件，但为我们提供了一种预览的方法

自定义模板

在scaffolds文件夹里可以看到，post，page，draft三个md文件，我们创建时是通过这三个模板创建的。我们也可以自定义模板。

文件名称

我们用 hexo new xxx 时，xxx是文章名称，即在文件里面头部的Front-matter里的title。文件的名称默认页是跟文章名称一样。

可以设置 _config.yml 文件里的 new_post_name 设置。规则如下

Placeholder 描述
:title 标题（小写，空格将会被替换为短杠）
:year 建立的年份，比如， 2015
:month 建立的月份（有前导零），比如， 04
:i_month 建立的月份（无前导零），比如， 4
:day 建立的日期（有前导零），比如， 07
:i_day 建立的日期（无前导零），比如， 7

例如 :year-:month-:day-:title.md
